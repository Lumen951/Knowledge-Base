# 📚 Obsidian 知识库使用指南

## 🎯 核心原则
- **简单实用**：标签扁平化，命名标准化
- **结构化**：严格按PARA方法分类
- **连接性**：每篇笔记至少链接1个相关笔记
- **可持续**：建立固定工作流，定期回顾

---

## 📁 文件夹结构

### 主目录结构
```
00_Inbox/          # 临时收集，每日清空
01_Notes/          # 日常快速笔记，不归类的内容
02_Projects/       # 有期限、有目标的项目
03_Areas/          # 长期关注的领域（核心）
04_Resources/      # 参考资料库
05_Archives/       # 已完成内容归档
```

### 重点领域（03_Areas）
```
03_Areas/
└── 前沿研究/          # 研究方向
    ├── AI/
    ├── BCI/
    └── 软件工程/
```

---

## 📝 文件命名规则

### 标准格式
```
课堂笔记：2025-11-30 嵌入式系统 - RTOS入门.md
论文笔记：2024-Bao-LetMeTalk.md
研究笔记：BCI-脑电信号预处理方法.md
日常思考：2025-11-30 关于AI的思考.md
```

### 命名要点
- 日期在前，便于排序
- 用中横线分隔，不用空格
- 包含足够关键词，便于搜索

---

## 🏷️ 标签系统

### 内容类型（必选一个）
`#课堂笔记` `#论文笔记` `#研究笔记` `#技术文档` `#随想`

### 学科领域（按需选择）
`#AI` `#BCI` `#嵌入式` `#控制` `#自动化`

### 细分技术（可选）
`#深度学习` `#脑电信号` `#RTOS` `#传感器` `#信号处理`

### 状态管理
`#待整理` `#待深入` `#已完成` `#待实验`

### 使用规则
- 每篇笔记2-5个标签足够
- 优先使用现有标签，不随意创建新标签
- 标签用中文，简单易懂

---

## 📋 标准模板

所有笔记都应包含：

```yaml
---
tags: [选择标签]
date: 2025-MM-DD
status: 进行中/已完成/待整理
---
```

### 三大模板
1. **课堂笔记**：要点 + 详细内容 + 待解决问题
2. **论文笔记**：核心创新 + 方法概述 + 个人思考
3. **研究笔记**：研究动机 + 文献综述 + 实验计划

## 🔄 日常工作流

### 每日流程
1. **创建**：在00_Inbox创建当日笔记
2. **记录**：课堂/阅读时快速记录
3. **整理**：课后添加标签和链接
4. **归位**：睡前移入正确文件夹

### 每周流程
1. **回顾**：查看本周所有笔记
2. **连接**：补充笔记间的链接
3. **归档**：将完成的笔记移至对应目录
4. **清理**：清空00_Inbox

### 定期维护
- **每月**：检查标签使用情况，整理重复标签
- **每季度**：回顾Areas中的知识体系，更新MOC（知识地图）

---

## ⚡ 快速操作

### 创建新笔记
1. 选择对应模板
2. 填写YAML元数据
3. 至少链接1个相关笔记
4. 选择合适的标签

### 查找内容
- **按时间**：用calendar插件
- **按标签**：侧边栏标签面板
- **按关键词**：全局搜索
- **按关联**：图谱视图或反向链接

---

## 💡 最佳实践

### 避免事项
❌ 不要创建过多文件夹层级
❌ 不要过度细分标签
❌ 不要让笔记孤立存在
❌ 不要忘记定期回顾

### 推荐做法
✅ 每天清空Inbox文件夹
✅ 每篇笔记至少1个链接
✅ 标签保持在2-5个
✅ 用dataview创建索引页面
✅ 定期备份（已配置git）
