# 第一节课：从 0 到 1——手把手搭建你的 AI 驱动知识库

> **课程时间：** 2025-11-23 17:00
> **目标学员：** 对AI和知识管理感兴趣的学习者（无需编程基础）

---

## 一、课程定位与目标

### 1. 本节目标（What & Why）

学完这一节，你应该能做到：

- **认知升级**：理解「聊天窗口」是快餐，「本地知识库」是粮仓
- **环境搭建**：从零搭建 Obsidian，并在本地运行起第一个 AI CLI 工具
- **技能习得**：掌握 Markdown 核心语法，学会用终端（Terminal）和电脑对话
- **数据资产**：用 GitHub 托管你的知识，开启「版本控制」的人生

### 2. 开场引导（5–10 分钟）

#### 痛点唤醒

> "大家回想一下，上周你在 ChatGPT 或某个 AI 网页里聊出的精彩内容，现在还在吗？还能一秒钟找到吗？如果那个网页挂了，你的知识是不是也丢了？"

#### 现状描述

我们现在的知识像「散落的拼图」：
- 截图在相册
- 链接在微信收藏
- 笔记在备忘录
- 聊天记录、书签是不是散落一地？

#### 愿景描绘

我们要搭建一个**「第二大脑」**：
- 它不依赖网络（断网也能用）
- 它属于你（数据在你硬盘里）
- 它能和 AI 协作（AI 帮你写、帮你改）

#### 全景图展示（建议板书或 PPT）

```
输入（Input）：终端命令/灵感碎念
         ↓
处理（Process）：AI CLI 工具（清洗、结构化、扩展）
         ↓
存储（Storage）：Obsidian（Markdown 文件）
         ↓
备份与版本（Version）：Git + GitHub
```

**核心工作流：** `终端命令 → AI 处理 → 生成文件（MD/LaTeX）→ Git 管理 → 持续迭代`

## 二、核心概念：为什么是「AI + 知识库」和什么是 CLI

### 1. 为什么要「AI + 知识库」？

#### 超越聊天记录

- **聊天是流动的（Flow），知识库是凝固的（Stock）**
- AI 聊天记录不适合长期管理：难搜索、难结构化、难复用
- AI 的回答通常需要**二次加工**才能变成知识

#### 知识库的三大特性（类比法）

- **可链接**：像维基百科一样，知识点之间有超链接，形成网络
- **可演化**：笔记不是一次写完的，而是像软件一样不断升级版本
- **可协作**：你的笔记是标准的文本文件，不仅人能看，AI 也能看，未来的程序也能读取

#### AI 的新角色

- **以前**：它是陪聊网友
- **现在**：它是你的**「数字图书管理员」**
  - 你给它一堆乱七八糟的素材
  - 命令它：「帮我整理成 Markdown 格式存好」
  - 从「聊天对象」升级为「写作合作者」和「整理助手」

### 2. 什么是 CLI（命令行界面）？

#### 直观对比

| 形态 | 比喻 | 特点 |
|------|------|------|
| **GUI（图形界面）** | 像去餐厅点菜，看着菜单（图标/按钮）指指点点 | 直观，但只能做菜单上有的菜 |
| **CLI（命令行）** | 像直接走进后厨跟大厨说话 | **精准、高效、可组合** |

**CLI 举例：**
```bash
"我要一份蛋炒饭，不要葱，加双倍蛋，三分钟后做好"
```

#### AI 产品形态光谱

| 形态 | 代表产品 | 适用场景 |
|------|---------|---------|
| **Chatbot** | ChatGPT网页版 | 头脑风暴、临时对话 |
| **Automation** | Make/Zapier | 固定流程、工作流自动化 |
| **CLI** | Gemini CLI/Claude Code | **开发者和极客，用一行代码调用 AI，直接操作本地文件** |

**给 CLI 下一个学生能听懂的定义：**
> "用命令行操控程序的方式，本质是用一串文本指令来告诉电脑你要干什么"

## 三、Obsidian 入门：从零搭建你的知识库

### 1. 安装与哲学

#### 操作步骤

1. 下载安装 Obsidian（官网：https://obsidian.md）
2. 创建第一个 Vault（仓库）

**命名建议：**
- `knowledge-base` 或 `brain-dump` 或 `ai-knowledge-base`

**位置建议：**
- 放在非系统盘，或者专门的一个 `Documents/Wiki` 目录下

#### 核心哲学（Highlight）

> "Obsidian 只是一个浏览器。你看到的每一个页面，本质上都是你硬盘里的一个 `.md` 文件。哪怕 Obsidian 倒闭了，你的笔记用记事本也能打开。"

**软件特点：**
- 所有内容都是本地 Markdown 文件
- 不锁定在平台里，可以配合 Git、VS Code 等工具
- 插件生态非常丰富（模板类、图谱类、任务类插件）

### 2. 文件夹结构（简单即正义）

不要过度设计，建立三个核心文件夹：

```
ai-knowledge-base/
├── inbox/        # 收件箱：所有新东西先丢这里，不分类
├── notes/        # 原子笔记：整理好的、单一知识点的笔记
└── projects/     # 项目：针对具体任务（如「本课程学习」）的笔记集合
```

**练习：** 创建第一篇笔记 `今天的 AI 课笔记.md`

### 3. Markdown 极速入门（实操环节）

#### 概念

Markdown 是一种「把排版指令直接写在文字里」的轻量级标记语言。

#### 常用语法速查表（带着学生在 Obsidian 里敲一遍）

| 功能 | 语法 | 效果 |
|------|------|------|
| **标题** | `# 大标题`<br>`## 二级标题` | 层级化的标题 |
| **强调** | `**重点**`<br>`*次重点*` | **粗体** 和 *斜体* |
| **列表** | `- 观点1`<br>`1. 步骤1` | 无序和有序列表 |
| **引用** | `> 鲁迅说过……` | 引用块 |
| **代码** | `` `code` ``<br>````python`<br>`print("hello")`<br>```` | 行内代码和代码块 |
| **双链（灵魂功能）** | `[[另一篇笔记]]` | 内部链接 |
| **任务清单** | `- [ ] 待办事项` | 可勾选的任务 |

#### 课堂练习

**任务：**
- 新建笔记 `Lesson1_Note.md`
- 把黑板上的课程大纲抄下来
- 必须用到：标题、列表和加粗

**提示：** 完成后把「第一节课大纲」用 Markdown 整理成一篇笔记

## 四、编程与命令行：为 AI 工作流打底

### 1. 编程语言的本质

#### 通俗解释

- **机器语言**：纯数字指令（0和1），只有电脑看得懂
- **汇编语言**：把数字换成简单的缩写单词，人类稍微能看懂一些
- **高级语言**：Python、JavaScript 等，更接近人类语言

**关键结论：**
- 编程语言是「翻译官」
- Python/JavaScript 是高级翻译官，它们听懂人类的逻辑，翻译给电脑听
- 你以后在终端里敲的命令，本质上也是在给电脑下指令
- AI CLI 工具就是「用命令行形式去调用大模型」

#### Shell/终端是什么？

- 它是电脑的「驾驶舱」
- 你在这里输入指令，直接操控操作系统内核

**对比：**
- **图形界面**：点点点
- **终端**：说清楚你要做什么
- **AI CLI**：用终端话语去让 AI 做事

### 2. VS Code + 终端初体验

#### 安装

1. 安装 VS Code（官网：https://code.visualstudio.com）
2. 选择 "Open Folder" 打开刚才的 Obsidian 文件夹

#### 调用终端

- 快捷键：`` Ctrl + ` ``（Mac/Win 通用）

#### 三大咒语（基础命令）

| 命令 | 全称 | 作用 |
|------|------|------|
| `pwd` | Print Working Directory | 我在哪？ |
| `ls` | List | 这里有什么？ |
| `cd` | Change Directory | 我要去哪？ |

#### 体验环节

**任务：**
1. 让学生在终端里输入 `ls`
2. 看到自己刚才创建的 `.md` 文件出现在黑底白字的屏幕上

**点评：**
> "恭喜你们，刚刚绕过了漂亮的图形界面，直接触碰到了文件的灵魂。"

## 五、安装并测试 Gemini CLI / Claude Code

### 1. 安装环境（难点攻克）

> **注意：** 这是最容易卡住的地方。建议提前准备好 Python 或 Node.js 环境包，或者使用最简单的安装脚本。

#### 两种工具选择

| 工具 | 安装方式 | API获取 |
|------|---------|--------|
| **Gemini CLI** | `pip install -q -U google-generativeai` | Google AI Studio |
| **Claude Code** | 官方安装脚本 | Anthropic 官网 |

#### 以 Google Gemini API 为例（假设使用 Python）

1. 提前申请 API Key（Google AI Studio）
2. 安装库：
   ```bash
   pip install -q -U google-generativeai
   ```

#### 配置安全

**演示如何设置环境变量：**
- 创建 `.env` 文件
- 或者在终端临时 `export API_KEY="your-key"`

**安全警示：**
> "API Key 是你连接 AI 大脑的钥匙，也是你的信用卡（如果付费）。**绝对不能**截图发朋友圈，也不能上传到公开的代码仓库。"

**重要提醒：**
- Key 不要公开
- 不要上传到公开仓库
- 后续会用 `.gitignore` 排除

### 2. 第一次「人机命令行对话」

#### 场景设计

我们不要在网页里聊天，我们要在终端里让 AI 写文件。

#### 指令示例（假设已配置好名为 `ask-ai` 的别名/脚本）

**基础对话：**
```bash
ask-ai "请用一句话解释什么是量子力学"
```

**进阶指令（流向文件）：**
```bash
ask-ai "请总结 Markdown 的 5 个核心语法，生成一篇简短的教程" > markdown_tutorial.md
```

#### 见证奇迹

**步骤：**
1. 执行完命令后，回到 Obsidian
2. 你会发现 `markdown_tutorial.md` 文件**凭空出现**了！

**关键点：**
> 这就是自动化的雏形——用命令行生成内容，直接存入知识库。

**强化「核心工作流」印象：**
```
命令行指令 → AI 输出 → 粘贴进 Markdown → 进入知识库
```

#### 课堂练习

**任务：**
- 输入一条命令，请 AI 用三句话总结今天学了什么
- 将输出复制回 Obsidian，保存为一篇总结笔记

## 六、GitHub 入门与 GitHub Desktop

### 1. 为什么要用 GitHub？

| 功能 | 说明 |
|------|------|
| **时光机** | 写错了？误删了？GitHub 能帮你回退到昨天的版本 |
| **云同步** | 家里电脑写一半，推送到云端，去学校拉取下来继续写 |
| **开源简历** | 你的学习记录就是你最好的简历 |

**对学生的意义：**
- 软件仓库与版本管理
- 存放你的学习笔记、论文、项目文档
- 公开展示自己的项目和学习轨迹
- 简历上可以写的东西

### 2. 极简工作流（GitHub Desktop 版）

#### 注册与安装

1. 注册 GitHub 账号（github.com）
2. 下载安装 GitHub Desktop

#### 初始化仓库

- 在 GitHub Desktop 中 "Add Local Repository"
- 选中你的 Obsidian 文件夹（如 `ai-knowledge-base`）

#### 忽略文件（.gitignore）

**重要步骤：** 教大家新建 `.gitignore` 文件

**必须包含的内容：**
```gitignore
.DS_Store
.obsidian/workspace
.obsidian/workspace.json
.env
*_API_KEY.txt
```

**说明：**
- 避免同步无关的配置信息
- **更重要**：确保包含 API Key 的文件被忽略！

### 3. 第一次 Commit（提交）

#### 概念区分

| 操作 | 说明 |
|------|------|
| **Save（保存）** | `Ctrl+S`，只是存到硬盘，可能会被覆盖 |
| **Commit（提交）** | 像玩游戏存盘点，给当前状态拍个照，写上备注 |
| **Push（推送）** | 把照片上传到云端服务器 |

#### 实操步骤

1. 在 GitHub Desktop 中识别到当前文件变化
2. 在 Summary 栏输入：
   ```
   feat: init my knowledge base
   ```
   或
   ```
   chore: init ai knowledge base
   ```
3. 点击 "Commit to main"
4. 点击 "Push origin"

#### 验收

- 打开 GitHub 网页端
- 看到自己的笔记静静地躺在网页上
- 点开网页，让大家看到自己的在线知识库仓库

## 七、第一节课收尾与课后作业

### 1. 课堂小复盘（Checklist）

请检查你是否完成：

- [ ] 你的电脑上有了 Obsidian 仓库
- [ ] 你学会了写 `# 标题` 和 `[[链接]]`
- [ ] 你的终端（Terminal）不再可怕，你知道怎么用 `ls` 看文件
- [ ] 你成功用命令行调用 AI 生成了一个文件（这是最酷的一步！）
- [ ] 你的知识库已经安全地托管在 GitHub 上了

**复盘三个关键「第一次」：**
- 第一个本地知识库
- 第一次命令行调用 AI
- 第一次 Git 提交

### 2. 课后作业（Actionable）

#### 作业方案 A：《我的数字花园规划书》

**要求：**
1. 在终端使用 AI CLI 命令，让 AI 帮你生成文章大纲
2. 将大纲重定向（`>`）到一个 `.md` 文件中
3. 在 Obsidian 里根据大纲完善内容（不少于 200 字）
4. 使用 GitHub Desktop 提交并推送到远程仓库
5. **提交方式**：发送你的 GitHub 仓库链接给老师/助教

#### 作业方案 B：《个人知识管理现状与规划》

**要求：**
- 每人用 Obsidian 写一篇「个人知识管理现状与规划」
- 用 AI CLI 帮你把这篇文章润色成更清晰的结构
- 完成后提交到 GitHub 仓库，再做一次 commit

### 3. 下一步预告

> "今天我们搭好了房子（知识库）并请来了管家（CLI）。下节课，我们要教管家如何**自动阅读**你的笔记，并帮你生成新的见解。我们将进入 AI 自动化信息处理的世界。"

**下节课内容预告：**
- 设计你知识库的文件夹结构与笔记模板
- 用 Git 做日常版本管理和简单分支协作
- 利用 Gemini CLI / Claude Code 批量整理「多信息源」资料
- 形成一个「从信息输入到知识沉淀」的标准工作流

## 给讲师的备课 Tips

### 1. 环境一致性

如果学生设备差异大（Win/Mac），建议：
- 提前准备好一份《课前环境配置文档》
- 涵盖：Git 安装、VS Code 安装、Python/Node 安装
- 提供统一的安装脚本

### 2. API Key 额度

**成本提醒：**
- **Gemini**：Google AI Studio 目前有免费额度
- **Claude/OpenAI**：需提醒学生可能有微小成本
- **或者**：由老师提供统一的 Key（注意限流）

### 3. CLI 工具选择

**对于初学者，推荐使用：**
- `llm`（Simon Willison 开发的工具）
- 或简单的 Python 脚本包装器
- 比官方 SDK 更容易上手

**示例：**
```bash
pip install llm
llm "hello"
```
这比写 Python 代码调用 SDK 直观得多。

### 4. 时间分配建议

| 环节 | 时间 | 备注 |
|------|------|------|
| 开场引导 | 10分钟 | 建立认知，唤醒痛点 |
| 核心概念讲解 | 15分钟 | AI+知识库、CLI介绍 |
| Obsidian 实操 | 20分钟 | 安装、创建笔记、Markdown练习 |
| 命令行基础 | 15分钟 | VS Code、终端、基础命令 |
| AI CLI 安装与测试 | 30分钟 | **重点环节**，容易卡壳 |
| GitHub 入门 | 20分钟 | 注册、初始化、第一次提交 |
| 收尾与作业 | 10分钟 | 复盘、布置作业 |

### 5. 常见问题预案

**Q1：Python/Node 环境安装失败**
- 准备好便携版或 Docker 镜像
- 或使用线上 Colab/Replit

**Q2：API Key 获取困难**
- 提前准备好图文教程
- 或提供临时共享 Key（限流保护）

**Q3：Git 推送失败（网络问题）**
- 准备好 VPN/镜像源配置
- 或暂时跳过，课后补充
