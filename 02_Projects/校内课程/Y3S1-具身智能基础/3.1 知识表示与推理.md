# 3.1 基本概念

知识的特性：
1. 相对正确性
2. 不确定性
	1. 随机性：关于概率
	2. 模糊性：关于定义不清
	3. 不完全性：关于信息缺失
	4. 经验性：关于经验法则的局限性




# 3.2 一阶谓词逻辑表示法

一阶谓词逻辑是命题逻辑的扩展，是AI中**基础且严谨**的知识表示形式化方法。

## 一、核心要素
- **个体（常量）**：具体对象，如 `小明`、`5`
- **谓词**：表示属性或关系，如 `人(小明)`、`在(书, 桌子)`
- **变量**：表示不确定个体，如 `x`、`y`
- **函数**：将个体映射到个体，如 `父亲(小明)`
- **连接词**：`¬`（非）、`∧`（与）、`∨`（或）、`→`（蕴含）、`↔`（等价）
- **量词**：
  - `∀`（全称）："对于所有..."
  - `∃`（存在）："存在..."

## 二、表示示例
| 自然语言 | 一阶谓词逻辑 |
| :--- | :--- |
| 小明是人 | `人(小明)` |
| 所有学生都学习 | `∀x(学生(x) → 学习(x))` |
| 存在一个红色的球 | `∃x(球(x) ∧ 红色(x))` |
| 如果天下雨，那么地湿 | `下雨(天) → 湿(地)` |

## 三、推理规则
- **全称实例化**：`∀x P(x) ⊢ P(a)`
- **存在实例化**：`∃x P(x) ⊢ P(c)` （c为新常量）
- **假言推理**：`P → Q, P ⊢ Q`



# 3.3 产生式表示法

产生式表示法（Production Rule Representation）是一种基于“**条件-动作**”对的知识表示方法，广泛用于专家系统和规则引擎。

## 一、基本结构
一个产生式通常表示为：
```
IF <条件> THEN <动作/结论>
```
或
```
前提 → 结论
```

## 二、核心组成
1. **事实（事实库）**
   - 已知的断言或数据，如：`温度 > 38℃`、`病人咳嗽`

2. **规则（规则库）**
   - 条件与结论的集合，如：
     ```
     IF 发烧 AND 咳嗽 THEN 可能患感冒
     IF 账户余额 < 0 THEN 发送透支警告
     ```

3. **推理机**
   - 控制规则匹配与执行的引擎

## 三、推理方式
| 推理方向 | 过程 | 应用场景 |
| :--- | :--- | :--- |
| **正向推理**（数据驱动） | 从已知事实出发，匹配规则前提，推出新结论 | 诊断系统、监控系统 |
| **反向推理**（目标驱动） | 从目标假设出发，寻找支持该假设的规则和事实 | 规划系统、证明系统 |
| **混合推理** | 结合正向与反向推理 | 复杂问题求解 |

## 四、表示示例
**医疗诊断规则**：
```
规则1: IF 发热 AND 咳嗽 THEN 疑似感冒
规则2: IF 疑似感冒 AND 流鼻涕 THEN 感冒(置信度0.8)
规则3: IF 感冒 AND 体温>39℃ THEN 建议就医
```

**动物识别规则**：
```
IF 有毛发 THEN 哺乳动物
IF 哺乳动物 AND 有蹄 THEN 蹄类动物
IF 蹄类动物 AND 有长颈 THEN 长颈鹿
```

## 五、冲突消解策略
当多条规则条件满足时，常用策略：
1. **专一性排序**：选择条件更具体的规则
2. **数据排序**：使用最新或最相关的数据
3. **规则排序**：按优先级或顺序执行
4. **上下文限制**：只在特定上下文中激活



# 3.4 框架表示法

框架表示法（Frame Representation）是一种基于**结构化对象**的知识表示方法，模拟人类对典型情境的认知结构。它通过“框架”来描述一类对象或情境的通用属性。

## 一、基本概念
- **框架**：描述一类对象或情境的数据结构
- **槽**：框架的属性或特征
- **侧面**：槽的进一步细化，描述属性的各个方面
- **值**：槽或侧面的具体取值

## 二、框架结构
```
框架名: <框架名称>
  槽1: <值/默认值>
    侧面1: <值>
    侧面2: <值>
  槽2: <值/默认值>
  槽3: <值/默认值>
    ...
```

## 三、表示示例

**1. 学生框架**
```
框架名: 学生
  姓名: (默认: 未知)
  学号: (必须)
  年龄: (默认: 18)
    范围: (18-30)
  专业: (默认: 未定)
  成绩: (默认: 空表)
```

**2. 教室框架**
```
框架名: 教室
  教室号: (必须)
  容量: (默认: 50)
    单位: 人
  设备: (默认: [黑板, 投影仪])
  当前课程: (默认: 无)
  上课时间: (默认: 空表)
```

**3. 继承关系示例**
```
框架名: 交通工具
  动力类型: (默认: 未知)
  载客量: (默认: 1)
  速度范围: (默认: 0-200)

框架名: 汽车
  父框架: 交通工具
  动力类型: (默认: 燃油)
  轮子数: (默认: 4)
  品牌: (默认: 未知)

框架名: 电动汽车
  父框架: 汽车
  动力类型: (默认: 电动)
  电池容量: (默认: 60)
    单位: kWh
```

## 四、核心特性

1. **继承性**
   - 子框架继承父框架的槽和默认值
   - 支持多重继承

2. **默认值**
   - 槽可以有默认值，当信息缺失时使用
   - 默认值可被具体值覆盖

3. **过程附件**
   - **if-needed**：需要值时触发
   - **if-added**：添加值时触发
   - **if-removed**：删除值时触发
   ```
   槽: 年龄
     值: 20
     if-needed: 计算年龄(出生日期)
     if-added: 验证年龄范围(值)
   ```

4. **约束**
   - 值类型约束：整数、字符串、列表等
   - 值域约束：范围限制
   - 依赖约束：槽间依赖关系

## 五、推理机制

1. **匹配**
   - 将当前情境与框架库中的框架匹配
   - 选择最合适的框架

2. **槽填充**
   - 使用默认值填充缺失信息
   - 通过过程附件获取或计算值

3. **继承推理**
   - 沿继承链查找属性值
   - 处理属性冲突（最近优先）


# 3.5 语义网络表示法

语义网络（Semantic Network）是一种用**图结构**表示知识的方法，通过节点和边来表示概念、实体及其之间的关系。它模拟人类联想记忆的结构，直观地展示知识间的联系。

## 一、基本组成

- **节点**：表示概念、对象、实体或属性
  - 如：`人`、`小明`、`红色`、`动物`
- **边（弧）**：表示节点间的关系
  - 如：`是`、`有`、`位于`、`属于`
- **标记**：边上的标签说明关系类型

## 二、基本关系类型

| 关系类型 | 符号/表示 | 示例 |
| :--- | :--- | :--- |
| **实例关系** | `ISA` 或 `是` | `小明` → `是` → `学生` |
| **分类关系** | `AKO` (A Kind Of) | `汽车` → `是一种` → `交通工具` |
| **属性关系** | `有`、`属性` | `鸟` → `有` → `翅膀` |
| **组成部分** | `Part-of` | `轮子` → `是...的一部分` → `汽车` |
| **位置关系** | `位于`、`在...中` | `书` → `位于` → `桌子` |
| **时间关系** | `之前`、`之后` | `早餐` → `在...之前` → `午餐` |
| **因果关系** | `导致`、`引起` | `下雨` → `导致` → `地湿` |

## 三、表示示例

**简单示例**（图形表示）：
```
      是
小明 ────→ 学生
      学习于
       ↓
      北京大学
```

**文本表示**（三元组形式）：
```
(小明, 是, 学生)
(学生, 学习于, 北京大学)
(北京大学, 位于, 北京)
(北京, 是, 城市)
```

**动物分类示例**：
```
鸟 ──是一种──→ 动物
鸟 ──有──→ 翅膀
鸟 ──有──→ 羽毛
企鹅 ──是一种──→ 鸟
企鹅 ──生活于──→ 南极
企鹅 ──不能──→ 飞
```

## 四、推理机制

1. **继承推理**
   - 沿 `ISA` 或 `AKO` 链继承属性
   - 例：已知`鸟会飞`，`企鹅是鸟`，可推出`企鹅会飞`（需处理例外）

2. **匹配推理**
   - 在网络上匹配模式
   - 例：查找`所有会飞的动物`

3. **连接推理**
   - 通过路径连接相关概念
   - 例：`小明` → `学习` → `数学` → `是` → `学科`

| 特性 | 语义网络 | 框架表示 |
| :--- | :--- | :--- |
| **核心结构** | 图（节点+边） | 层次化槽结构 |
| **关系表示** | 显式边标签 | 隐含在槽结构中 |
| **继承机制** | 沿边传递 | 明确的父框架链接 |
| **形式化程度** | 较低 | 中等 |
| **可视化** | 优秀 | 一般 |
| **处理例外** | 困难 | 相对容易 |
